# 记账联动账户余额 - Trae Spec

## Overview
- **Summary**: 记账时选择账户后自动更新余额，支出减少余额，收入增加余额，转账在转出账户减少、转入账户增加。同时支持将支出记入折旧资产，自动创建折旧资产记录；折旧资产卖出时自动创建收入记录。
- **Purpose**: 确保账户余额与实际一致，支持折旧资产生命周期管理的记账端打通。
- **Target Users**: 所有记账用户

## Goals
- 记账时必须选择账户
- 支出减少账户余额，收入增加账户余额
- 转账在两个账户间转移金额
- 支出类型可开启"记入资产"开关
- 开关打开后展开折旧资产字段
- 记账成功同时创建折旧资产记录
- 折旧资产卖出时自动创建收入记录

## Non-Goals (Out of Scope)
- 不实现账户的创建/编辑（account-system）
- 不实现折旧资产的详细管理（depreciating-asset）

## Background & Context
记账联动是"记账省心"三角飞轮的核心功能，确保每笔记账自动更新账户余额。新增的"记入资产"开关让用户在记账当下即可创建折旧资产，实现折旧资产生命周期管理的闭环。

### 记账类型与账户余额变化

| 记账类型 | 账户余额变化 | 是否显示记入资产 |
|---------|------------|----------------|
| 支出 | 余额减少 | 是（打开开关可创建折旧资产） |
| 收入 | 余额增加 | 否 |
| 转账 | 转出减少、转入增加 | 否 |

## Functional Requirements

### FR-1: 账户选择
- 记账前必须选择账户
- 账户按类型分组展示
- 选择后显示账户图标+名称+余额
- 未选择时"记账"按钮禁用

### FR-2: 余额联动更新
- 支出：账户余额 = 原余额 - 支出金额
- 收入：账户余额 = 原余额 + 收入金额
- 转账：转出账户余额减少，转入账户余额增加

### FR-3: "记入资产"开关（仅支出类型）
- 位置：记账页金额输入区下方，选择账户之后
- 仅在"支出"类型下显示
- 开关默认关闭，打开后展开资产字段

### FR-4: 记入资产字段展开
- 品类标签：单选（9个选项）
- 折旧方法：直线法/双倍余额递减法
- 计划使用时长（月）
- 预期残值（元）
- 资产名称（默认取分类名）
- 选择品类后自动推荐折旧方法和默认参数

### FR-5: 记账成功同时创建折旧资产
- 记账请求成功后
- 如开启"记入资产"，同时调用创建折旧资产 API
- 折旧资产状态为"使用中"，当前价值=购入价

### FR-6: 折旧资产卖出自动创建收入
- 用户在资产详情页处置并选择"卖出"
- 填写卖出价格并确认
- 系统自动：
  - 更新资产状态为"已处置"
  - 在记账模块生成一笔收入记录
  - 金额=卖出价，分类=二手出售/资产变现
  - 备注自动填"出售：资产名称"

### FR-7: 编辑/删除记账回滚余额
- 编辑记账时计算差额，回滚原账户余额，更新新账户余额
- 删除记账时回滚账户余额

## Non-Functional Requirements
- **NFR-1**: 账户余额更新使用事务保证一致性
- **NFR-2**: 失败时回滚整笔记账
- **NFR-3**: 数字变化使用滚动动画，800ms
- **NFR-4**: 开关切换动画 200ms

## Constraints
- **Technical**: Vue3 + TypeScript + SCSS + Wot UI + UniApp
- **API**: RESTful 风格
- **Design**: 极简、纯白背景、SVG 线框图标、卡布里蓝体系

## Assumptions
- account-system 支持账户列表查询
- depreciating-asset 支持折旧资产创建和处置

## Acceptance Criteria

### AC-1: 记账时必须选择账户
- **Given**: 用户打开记账页
- **When**: 未选择账户
- **Then**: "记账"按钮禁用
- **Verification**: `human-judgment`

### AC-2: 支出减少余额
- **Given**: 账户余额¥1,000，用户记一笔¥200支出
- **When**: 记账成功
- **Then**: 账户余额变为¥800
- **Verification**: `integration-test`

### AC-3: 收入增加余额
- **Given**: 账户余额¥1,000，用户记一笔¥500收入
- **When**: 记账成功
- **Then**: 账户余额变为¥1,500
- **Verification**: `integration-test`

### AC-4: 转账转移余额
- **Given**: 账户A余额¥1,000，账户B余额¥500，用户转¥300
- **When**: 转账成功
- **Then**: 账户A变为¥700，账户B变为¥800
- **Verification**: `integration-test`

### AC-5: 支出类型显示"记入资产"开关
- **Given**: 用户在记账页
- **When**: 记账类型为"支出"
- **Then**: 显示"记入资产"开关
- **Verification**: `human-judgment`

### AC-6: 收入/转账类型不显示开关
- **Given**: 用户在记账页
- **When**: 记账类型为"收入"或"转账"
- **Then**: 不显示"记入资产"开关
- **Verification**: `human-judgment`

### AC-7: 开关打开展开字段
- **Given**: 用户打开"记入资产"开关
- **When**: 开关状态变为打开
- **Then**: 展开资产字段（品类、折旧方法、使用时长、残值、名称）
- **Verification**: `human-judgment`

### AC-8: 品类选择自动推荐折旧方法
- **Given**: 用户选择品类"手机"
- **When**: 品类选择完成
- **Then**: 折旧方法自动推荐"双倍余额递减"
- **Verification**: `human-judgment`

### AC-9: 记账成功同时创建折旧资产
- **Given**: 用户记一笔¥5,000支出，开启"记入资产"并填写完字段
- **When**: 记账成功
- **Then**: 
  - 支出记录创建成功
  - 折旧资产记录创建成功（状态=使用中，当前价值=¥5,000）
- **Verification**: `integration-test`

### AC-10: 折旧资产卖出自动创建收入
- **Given**: 某折旧资产当前价值¥4,500，用户卖出价格¥3,500
- **When**: 确认卖出
- **Then**: 
  - 资产状态变为"已处置"
  - 自动创建一笔¥3,500收入记录，分类为二手出售
- **Verification**: `integration-test`

### AC-11: 编辑记账回滚余额
- **Given**: 原支出¥200，账户余额¥800，用户改为¥300
- **When**: 编辑成功
- **Then**: 账户余额变为¥700
- **Verification**: `integration-test`

### AC-12: 删除记账回滚余额
- **Given**: 某支出记录¥200，账户余额¥800
- **When**: 删除成功
- **Then**: 账户余额变为¥1,000
- **Verification**: `integration-test`
