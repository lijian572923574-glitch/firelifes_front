# 折旧资产生命周期管理 - Trae Spec

## Overview
- **Summary**: 实现折旧资产从创建到处置的完整生命周期管理，包括记账时"记入资产"开关、资产列表/详情页、处置流程、两种折旧方法计算（直线法/双倍余额递减法）、月度折旧汇总
- **Purpose**: 帮助用户追踪可折旧资产（如手机、相机、鞋服等）的价值损耗，支持两种折旧方法，在资产处置时自动生成收入记录，并在月度报告中体现折旧损耗
- **Target Users**: 所有记账用户，尤其是有大件消费习惯的用户

## Goals
- 用户在记账时可通过开关将支出记入折旧资产
- 支持直线法和双倍余额递减法两种折旧方法
- 系统根据选择的折旧方法自动计算月折旧额和当前价值
- 用户可在资产列表查看所有折旧资产及其折旧进度
- 用户可对资产进行处置（报废/赠送/卖出），卖出时自动生成收入记录
- 月度 FIRE 报告中展示折旧损耗指标

## Non-Goals (Out of Scope)
- 不实现固定资产管理（单独的 spec）
- 不实现年数总和法（not supported）
- 不实现工作量法（not supported）
- 不实现资产折旧的手动批量操作
- 不实现折旧资产的批量导入/导出
- 不实现资产图片上传功能
- 不实现资产维修/保养记录

## Background & Context
当前记账应用只记录收支，无法追踪"买的东西还值多少"。用户对大件消费（如手机、相机）的价值损耗缺乏感知。通过在记账当下嵌入"记入资产"开关，降低用户记录门槛，实现资产全生命周期管理。

### 两种折旧方法

#### 直线法（年限平均法）
```
月折旧额 = (购入价 - 残值) ÷ 使用月数
特点：每月折旧固定不变
适用：相机、鞋服、家电、家具、包袋、运动、其他
```

#### 双倍余额递减法
```
月折旧率 = 2 ÷ 使用月数
当月折旧 = 期初账面价值 × 月折旧率
特点：折旧额逐月递减，前期多后期少
切换规则：当剩余使用期 <= 总使用期×40% 时，自动切换为直线法
适用：手机、电脑（默认推荐）
```

### 品类默认推荐表
| 品类 | 默认折旧方法 | 默认寿命 | 残值率 |
|------|------------|---------|--------|
| 手机 | 双倍余额递减 | 36个月 | 10% |
| 电脑 | 双倍余额递减 | 48个月 | 10% |
| 相机 | 直线法 | 60个月 | 15% |
| 家电 | 直线法 | 60个月 | 10% |
| 鞋服 | 直线法 | 24个月 | 10% |
| 家具 | 直线法 | 120个月 | 15% |
| 包袋 | 直线法 | 36个月 | 15% |
| 运动 | 直线法 | 36个月 | 5% |
| 其他 | 直线法 | 24个月 | 10% |

### 折旧计算示例

购入价 ¥10,000，计划使用 5年(60个月)，残值 ¥1,000

**直线法**：
- 月折旧 = (10000-1000)/60 = **¥150/月**（固定）
- 第1年末: ¥8,200 | 第3年末: ¥4,600 | 第5年末: ¥1,000

**双倍余额递减法**：
- 月折旧率 = 2/60 = 3.33%
- 第1月: ¥333 | 第12月: ¥222（逐月递减）
- 剩余24个月(≤60×40%)切换直线法
- 第1年末: ¥6,554 | 第3年末: ¥2,818 | 第5年末: ¥1,000

## Functional Requirements

### FR-1: 记账时"记入资产"开关
- 位置：记账页金额输入区下方，选择账户之后
- 仅在"支出"类型下显示（收入/转账不显示）
- 开关默认关闭，打开后展开资产字段
- 展开字段：
  - 品类标签：单选，选择后自动推荐折旧方法
  - 折旧方法：直线法 / 双倍余额递减法（选择品类后自动填充，可改）
  - 计划使用时长（月）：数字输入，默认品类推荐值
  - 预期残值（元）：数字输入，默认品类推荐值（购入价×残值率）
  - 资产名称：文本输入，默认取分类名，可改
- **折旧方法选定后不可更改**（与会计准则一致）
- 点击记账完成时，同时创建支出记录和折旧资产记录

### FR-2: 折旧资产列表页
- 入口：资产总览 → 折旧资产区块 → "查看全部"
- 顶部汇总：本月总折旧额、当前总价值
- Tab 筛选：使用中 / 已处置
- 排序：按价值降序（默认）/ 按购入时间倒序
- 列表项：资产图标 + 名称 + 品类标签 + 折旧方法标签 + 当前价值 + 折旧进度条
- 折旧进度条：已折旧 / (购入价 - 残值)，卡布里蓝填充
- 空态：显示引导"还没有折旧资产"

### FR-3: 折旧资产详情页
- 顶部：资产名称 + 品类标签 + 折旧方法标签 + 状态标签
- 大字显示当前价值（卡布里蓝）
- 价值曲线图：购入价 → 当前价值的折旧曲线
  - 直线法：曲线为直线
  - 双倍余额递减法：曲线为递减曲线
- 信息卡片：购入价、购入日期、计划使用时长、预期残值、月折旧额、已使用月数
- 操作按钮：
  - "调整价值"：手动修正当前价值
  - "处置"：进入处置流程
- 调整记录历史（可选展开）

### FR-4: 处置流程
- 处置方式选择：
  - **报废**：东西坏了/丢了，价值归0，无收入记录
  - **赠送**：送人了，价值归0，无收入记录
  - **卖出**：填写卖出价格
- 卖出确认后：
  - 资产价值归0，状态变为"已处置"
  - 自动在记账模块生成一笔收入记录（金额=卖出价，分类=二手出售）
  - 备注自动填"出售：资产名称"
- 处置记录保留在资产明细中

### FR-5: 自动折旧
- 触发时机：每月1日00:00自动执行
- 计算逻辑：
  - 对所有"使用中"且"已使用月数 < 计划使用月数"的资产执行折旧
  - 根据资产对应的折旧方法计算月折旧额
  - 双倍余额递减法到达剩余期≤总期×40%时切换直线法
  - 当前价值 = max(计算值, 残值)
- 价值不低于残值，到达使用期限后保持残值不变
- 折旧记录写入账户变动明细

### FR-6: 月度折旧汇总
- 月度 FIRE 报告中新增"本月折旧损耗"指标
- 展示：本月总折旧金额 + 折旧资产数量

## Non-Functional Requirements
- **NFR-1**: 页面切换动画流畅，列表加载不超过 300ms
- **NFR-2**: 折旧计算精确到分（2位小数）
- **NFR-3**: 资产数据本地缓存，离线可查看
- **NFR-4**: 支持 UniApp 兼容（iOS/Android/小程序/H5）
- **NFR-5**: 遵循卡布里蓝色彩体系，禁止黄色/暖色系

## Constraints
- **Technical**: Vue3 + TypeScript + SCSS + Wot UI + UniApp
- **API**: RESTful 风格
- **Design**: 极简、单色主色调、纯白背景、SVG 线框图标统一主色描边
- **Compatibility**: 不依赖 Phase 2/3 的其他功能
- **Depreciation**: 仅支持直线法和双倍余额递减法

## Assumptions
- 后端支持折旧资产的 CRUD API
- 后端支持定时任务（每月1日自动折旧）
- 记账模块已支持"生成收入记录"的能力
- 品类数据已存在，可获取默认参数

## Acceptance Criteria

### AC-1: 记账时记入资产
- **Given**: 用户在记账页选择支出分类
- **When**: 记账类型为"支出"
- **Then**: 金额输入区下方显示"记入资产"开关
- **Verification**: `human-judgment`

### AC-2: 开关打开后展开字段
- **Given**: 用户打开"记入资产"开关
- **When**: 开关状态变为打开
- **Then**: 展开资产字段（品类标签、折旧方法、计划使用时长、预期残值、资产名称）
- **Verification**: `human-judgment`

### AC-3: 品类选择自动推荐折旧方法
- **Given**: 用户选择品类标签（如"手机"）
- **When**: 品类选择完成
- **Then**: 折旧方法自动填入"双倍余额递减"，计划使用时长自动填入36个月，残值自动填入购入价×10%
- **Verification**: `human-judgment` + `unit-test`

### AC-4: 折旧方法选定后不可更改
- **Given**: 用户已选定折旧方法并保存
- **When**: 用户尝试修改折旧方法
- **Then**: 折旧方法字段变为只读，不可修改
- **Verification**: `human-judgment`

### AC-5: 记账完成同时创建资产
- **Given**: 用户填写完资产字段并点击记账
- **When**: 记账请求成功
- **Then**: 
  - 成功创建一笔支出记录
  - 成功创建一条折旧资产记录（状态=使用中，当前价值=购入价）
- **Verification**: `api-mock` + `integration-test`

### AC-6: 折旧资产列表展示
- **Given**: 用户进入折旧资产列表页
- **When**: 存在折旧资产数据
- **Then**: 
  - 顶部显示总当前价值和本月折旧额
  - 列表显示每个资产的名称、品类、折旧方法、当前价值、折旧进度条
- **Verification**: `human-judgment`

### AC-7: 直线法折旧计算正确
- **Given**: 某资产购入价¥10,000，残值¥1,000，计划60个月直线法，使用了12个月
- **When**: 系统计算当前价值
- **Then**: 月折旧 = ¥150，当前价值 = ¥8,200
- **Verification**: `unit-test`

### AC-8: 双倍余额递减法折旧计算正确
- **Given**: 某资产购入价¥10,000，残值¥1,000，计划60个月双倍余额递减法，使用了12个月
- **When**: 系统计算当前价值
- **Then**: 月折旧率 = 3.33%，第1月折旧 = ¥333，当前价值 ≈ ¥6,554
- **Verification**: `unit-test`

### AC-9: 双倍余额递减法自动切换直线法
- **Given**: 某资产购入价¥10,000，残值¥1,000，计划60个月双倍余额递减法，已使用38个月
- **When**: 系统计算当前价值（剩余22个月 ≤ 60×40% = 24个月）
- **Then**: 自动切换为直线法，按剩余价值计算月折旧
- **Verification**: `unit-test`

### AC-10: 资产处置-报废
- **Given**: 用户选择报废处置
- **When**: 确认报废
- **Then**: 
  - 资产状态变为"已处置"，处置方式="报废"
  - 当前价值归0
  - 不生成收入记录
- **Verification**: `integration-test`

### AC-11: 资产处置-卖出
- **Given**: 用户填写卖出价格¥3,500并确认
- **When**: 确认卖出
- **Then**: 
  - 资产状态变为"已处置"，处置方式="卖出"
  - 当前价值归0
  - 自动生成一笔收入记录（¥3,500，二手出售分类）
- **Verification**: `integration-test`

### AC-12: 卖出记录展示盈亏
- **Given**: 某资产当前价值¥4,500，用户以¥3,500卖出
- **When**: 查看处置记录
- **Then**: 显示"亏了 ¥1,000"
- **Verification**: `human-judgment`

### AC-13: 超过使用期限价值保持残值
- **Given**: 某资产计划60个月，已使用72个月，残值¥1,000
- **When**: 每月1日自动折旧
- **Then**: 当前价值保持为¥1,000，不继续折旧
- **Verification**: `unit-test`

### AC-14: 月度报告包含折旧损耗
- **Given**: 用户查看月度 FIRE 报告
- **When**: 报告生成中
- **Then**: 报告显示"本月折旧损耗：¥XXX，共X件资产"
- **Verification**: `integration-test`

### AC-15: 调整价值后重新计算
- **Given**: 某资产当前价值¥4,500，用户手动调整为¥4,800
- **When**: 确认调整
- **Then**: 
  - 当前价值更新为¥4,800
  - 记录调整历史（老值¥4,500 → 新值¥4,800）
- **Verification**: `integration-test`
