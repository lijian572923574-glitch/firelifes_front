# 固定资产管理 - Trae Spec

## Overview
- **Summary**: 实现固定资产（房产、车位、商铺等）的独立管理，包括添加/编辑、估值更新、估值历史追踪、关联负债账户、净权益计算、估值到期提醒
- **Purpose**: 帮助用户管理大额固定资产，追踪房产等资产的价值变化，计算净资产时自动关联房贷等负债
- **Target Users**: 有房产、车位、商铺等固定资产的用户

## Goals
- 用户可添加和管理固定资产（房产/车位/商铺/其他）
- 用户可更新固定资产估值并查看历史记录
- 用户可关联负债账户（房贷等）并自动计算净权益
- 超过6个月未更新估值时自动提醒
- 固定资产数据可接入资产总览

## Non-Goals (Out of Scope)
- 不实现折旧资产的自动折旧计算
- 不实现固定资产的处置流程（不同于折旧资产）
- 不实现固定资产的图片上传
- 不实现多币种固定资产

## Background & Context
折旧资产（手机、相机等）采用自动折旧计算，而固定资产（房产等）通常随时间增值或保持价值，采用手动估值更新的方式管理。固定资产可关联房贷等负债账户，净权益 = 当前估值 - 关联负债余额。

## Functional Requirements

### FR-1: 固定资产添加
- 入口：资产总览 → 固定资产区块 → "添加"
- 填写字段：
  - 资产名称（必填，最多50字符）
  - 品类（必填：房产/车位/商铺/其他）
  - 购入价（必填，数字）
  - 购入日期（必填，日期选择器）
  - 当前估值（必填，默认同购入价）
  - 关联负债账户（可选，选择负债类账户）
- 关联负债后自动计算净权益 = 当前估值 - 负债余额
- 保存后自动创建一条购入记录到估值历史

### FR-2: 固定资产列表页
- 入口：资产总览 → 固定资产区块 → "查看全部"
- 顶部汇总：总净权益、总估值
- 列表项：
  - 资产图标 + 名称
  - 品类标签
  - 净权益（大字）
  - 估值金额 + 增值率（绿色/红色/灰色）
  - 关联负债（如果有）
- 卡片底部显示"估值可能已过时"提示（如果超过6个月）
- 空态：显示引导"还没有固定资产"

### FR-3: 估值更新
- 入口：固定资产详情页 → "更新估值"按钮
- 显示当前估值
- 输入新估值
- 自动计算增值/贬值金额和比率
- 选择更新日期（默认当天）
- 保存后自动创建估值记录

### FR-4: 估值历史
- 入口：固定资产详情页
- 折线图展示估值变化趋势
- 列表展示每条估值记录
- 标记购入记录和当前记录
- 可查看每次估值的变更说明

### FR-5: 关联负债
- 入口：固定资产详情页 → "编辑" → "关联负债"
- 选择要关联的负债账户（仅负债类账户）
- 自动拉取账户当前余额
- 计算净权益 = 当前估值 - 负债余额
- 可随时解除关联

### FR-6: 估值到期提醒
- 触发条件：超过6个月未更新估值
- 显示位置：
  - 资产总览卡片底部
  - 详情页顶部
- 提醒文案："估值可能已过时，建议更新"

## Non-Functional Requirements
- **NFR-1**: 页面加载不超过 300ms
- **NFR-2**: 估值计算精确到元
- **NFR-3**: 支持 UniApp 兼容（iOS/Android/小程序/H5）
- **NFR-4**: 遵循卡布里蓝色彩体系

## Constraints
- **Technical**: Vue3 + TypeScript + SCSS + Wot UI + UniApp
- **API**: RESTful 风格
- **Design**: 极简、纯白背景、SVG 线框图标

## Assumptions
- 后端支持固定资产的 CRUD API
- 负债账户余额可实时查询
- 用户可关联多个固定资产到同一负债账户

## Acceptance Criteria

### AC-1: 添加固定资产
- **Given**: 用户点击添加固定资产
- **When**: 填写完必填字段并保存
- **Then**: 
  - 固定资产创建成功
  - 自动创建一条购入估值记录
  - 关联负债后自动计算净权益
- **Verification**: `integration-test`

### AC-2: 固定资产列表展示
- **Given**: 用户进入固定资产列表页
- **When**: 存在固定资产数据
- **Then**: 
  - 顶部显示总净权益和总估值
  - 列表显示每个资产的名称、净权益、估值、增值率
- **Verification**: `human-judgment`

### AC-3: 估值更新
- **Given**: 用户在详情页点击"更新估值"
- **When**: 输入新估值并确认
- **Then**: 
  - 估值更新成功
  - 净权益重新计算
  - 增值率重新计算
  - 估值历史新增一条记录
- **Verification**: `integration-test`

### AC-4: 关联负债
- **Given**: 用户关联一个负债账户
- **When**: 关联成功
- **Then**: 
  - 净权益 = 当前估值 - 负债余额
  - 资产卡片显示关联的负债金额
- **Verification**: `integration-test`

### AC-5: 估值到期提醒
- **Given**: 某固定资产超过6个月未更新估值
- **When**: 查看资产总览或详情页
- **Then**: 显示"估值可能已过时"提示
- **Verification**: `human-judgment`

### AC-6: 增值率计算
- **Given**: 某资产购入价¥1,600,000，当前估值¥2,000,000
- **When**: 系统计算增值率
- **Then**: 增值率 = +25%
- **Verification**: `unit-test`

### AC-7: 净权益为负
- **Given**: 某资产当前估值¥1,500,000，关联负债¥1,800,000
- **When**: 系统计算净权益
- **Then**: 净权益 = -¥300,000（显示红色）
- **Verification**: `unit-test`
