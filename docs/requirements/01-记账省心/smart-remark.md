## 功能概述
在用户输入备注时，根据历史记账记录自动联想推荐。当用户选择某个分类后，优先展示该分类下最近使用的备注，减少重复输入，提升记账效率。

## 用户故事
作为用户，我在给"早餐"分类记账时，经常输入"麦当劳"或"永和大王"等商家名。希望系统能记住我的输入习惯，在我开始输入时自动显示这些历史备注，让我一键选择而非重复打字。

## 交互设计

### 页面结构
```
备注输入区域
┌────────────────────────────────────┐
│  📝 备注                            │
│  ┌─────────────────────────────┐   │
│  │ [输入框]                     │   │
│  └─────────────────────────────┘   │
│                                     │
│  [联想区域 - 输入后显示]             │
│  ┌─────────────────────────────┐   │
│  │ 麦 ↓                        │   │
│  ├─────────────────────────────┤   │
│  │ ▸ 麦当劳                     │   │
│  │ ▸ 麦当劳早餐套餐               │   │
│  │ ▸ 麦当劳优惠券                 │   │
│  └─────────────────────────────┘   │
└────────────────────────────────────┘
```

### 交互流程
1. 用户点击备注输入框
2. 输入字符后（≥1个字符），触发联想查询
3. 显示最多5条联想结果
4. 用户可点击选择或继续输入
5. 选择后自动填充到输入框
6. 输入框失焦或提交后，隐藏联想区域

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 默认 | 页面加载/无输入 | 不显示联想区域 |
| 联想中 | 输入字符 | 显示 loading 状态 |
| 有结果 | 查询返回 | 显示联想列表 |
| 无结果 | 查询返回空 | 隐藏联想区域 |
| 已选择 | 点击联想项 | 填充输入框 |

## 数据结构

### 新增接口
```typescript
// GET /api/remarks/suggestions - 获取备注联想
interface GetRemarksSuggestionsRequest {
  categoryId?: string;     // 当前分类ID，可选
  keyword: string;        // 搜索关键词
  limit?: number;         // 返回数量，默认5
}

interface GetRemarksSuggestionsResponse {
  code: number;
  data: {
    suggestions: RemarkSuggestion[];
  };
}

interface RemarkSuggestion {
  remark: string;         // 备注内容
  count: number;           // 使用次数（用于排序）
  lastUsedAt: string;      // 最后使用时间
}
```

## 边界情况
1. **空输入**：不触发联想
2. **网络错误**：静默失败，不影响正常输入
3. **特殊字符**：过滤后搜索
4. **超长备注**：输入框限制100字符
5. **频繁输入**：防抖处理，300ms 后再查询
