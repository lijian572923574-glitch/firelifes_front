## 功能概述
在金额输入区显示千分位格式（如 12,345.67），提升大金额的可读性。内部存储仍为纯数字字符串，提交时格式化为标准金额格式。

## 用户故事
作为用户，当我输入较大金额（如 123456）时，希望能自动显示为 123,456，方便我快速识别金额大小，避免输入错误。

## 交互设计

### 页面结构
```
金额输入显示区
┌────────────────────────────────────┐
│                                     │
│            ¥ 123,456.78            │ ← 千分位格式显示
│                                     │
│  ────────────────────────────────  │
│                                     │
│           [0.00]                   │ ← 实际输入位置（透明）
└────────────────────────────────────┘
```

### 交互流程
1. 用户点击金额输入区域
2. 弹出数字键盘
3. 用户输入数字
4. 实时显示千分位格式化后的金额
5. 提交时移除千分位，存储纯数字

### 格式化规则
| 输入值 | 显示值 |
|--------|--------|
| 0 | ¥ 0.00 |
| 12 | ¥ 12.00 |
| 1234 | ¥ 1,234.00 |
| 123456 | ¥ 123,456.00 |
| 123456.5 | ¥ 123,456.50 |
| 123456.78 | ¥ 123,456.78 |

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 空 | 无输入 | 显示 ¥ 0.00 |
| 输入中 | 输入数字 | 实时千分位格式化 |
| 退格 | 删除数字 | 重新格式化 |
| 小数点 | 输入小数 | 保留两位小数 |

## 数据结构

### 格式化工具函数
```typescript
// 金额格式化
function formatAmount(value: string | number): string {
  // 1. 转换为字符串
  // 2. 移除非数字字符
  // 3. 保留两位小数
  // 4. 添加千分位分隔符
  return '¥ ' + value.toLocaleString('zh-CN', {
    minimumFractionDigits: 2,
    maximumFractionDigits: 2
  });
}

// 金额解析（提交时）
function parseAmount(formattedValue: string): string {
  // 移除货币符号和分隔符
  return formattedValue.replace(/[¥,]/g, '');
}
```

## 边界情况
1. **超长金额**：限制最大输入12位整数
2. **精度问题**：浮点数运算时保持精度
3. **粘贴内容**：过滤非数字字符
4. **空值提交**：提示输入金额
5. **负数**：不允许负数输入
