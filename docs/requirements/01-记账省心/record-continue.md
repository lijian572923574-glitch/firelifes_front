## 功能概述
记账成功后不跳转至明细页，而是弹出"继续记一笔"确认卡片。用户可以选择继续记账或等待3秒后自动跳转。这个功能简化了连续记账的场景，减少用户反复进出的操作步骤。

## 用户故事
作为用户，我在记录多笔同类支出（如一天内的多笔零食开销）时，希望能够快速连续记账，而不用每次都返回再进入记账页，以提升记账效率。

## 交互设计

### 页面结构
```
记账页
├── Header (时间 + 账户选择)
├── 金额输入区
├── 分类选择器
├── 备注输入
└── 提交按钮
    │
    ▼ (提交成功后)
┌─────────────────────┐
│   记账成功确认卡片    │
│  ┌───────────────┐  │
│  │  [分类图标]    │  │
│  │  - ¥68.00     │  │
│  │  记账成功      │  │
│  └───────────────┘  │
│                     │
│  [继续记一笔]  [去明细页] │
└─────────────────────┘
```

### 交互流程
1. 用户完成记账输入，点击提交
2. 提交成功后，显示记账成功确认卡片（2秒后消失）
3. 卡片底部显示两个按钮：
   - "继续记一笔"：留在当前页，重置金额和备注，保留分类选择
   - "去明细页"：立即跳转明细页
4. 如果用户3秒内无操作，自动跳转明细页
5. 点击"继续记一笔"后，清空金额输入框和备注输入框，保持分类选择不变

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 默认 | 页面加载 | 显示正常记账表单 |
| 提交中 | 点击提交 | 显示 loading |
| 成功 | 提交返回成功 | 显示确认卡片，开始3秒计时 |
| 继续记账 | 点击"继续记一笔" | 清空金额/备注，关闭卡片 |
| 跳转明细 | 点击"去明细页"或3秒超时 | 跳转 detail/index |

## UI 设计规范

### 布局
- 确认卡片：居中弹出，高度 200rpx，宽度 80%
- 卡片内图标：64rpx × 64rpx
- 按钮行：底部 20rpx 留白，按钮高度 64rpx

### 颜色
- 卡片背景：#FFFFFF
- 卡片阴影：rgba(0, 191, 255, 0.2)
- 按钮主色：#00BFFF
- 按钮次色：#999999（去明细页按钮）

### 字体
- 金额：48rpx，#333333，加粗
- 按钮文字：28rpx
- 成功提示：24rpx，#999999

### 动效
- 卡片弹出：scale 0.8→1 + opacity 0→1，时长 200ms，ease-out
- 卡片消失：opacity 1→0，时长 150ms
- 3秒倒计时：进度条从右到左，颜色 #00BFFF

## 边界情况
1. **提交失败**：不显示确认卡片，显示错误提示，保持当前表单状态
2. **快速连续提交**：在 loading 状态下禁用提交按钮，防止重复提交
3. **3秒内跳转其他页面**：取消倒计时，清理计时器
4. **切换到其他 Tab**：卡片关闭，不影响原有跳转逻辑
5. **App 被切到后台**：卡片保持显示，恢复前台后继续计时
