## 功能概述
支持设置周期性自动记账（每天/每周/每月），适用于房租、工资、定投等固定收支场景。到期自动创建记录并通知用户确认。VIP 功能（当前可先做UI，功能预留）。

## 用户故事
作为用户，我每月1号需要记录房租支出，每年1月需要记录保险费。这些固定支出我不想每次手动输入，希望设置自动记账模板，到期自动生成记录，我只确认即可。

### 交互流程
1. 用户进入"自动记账"页面
2. 点击"创建自动记账"
3. 填写模板信息（类型/分类/金额/备注/账户/周期）
4. 保存模板
5. 系统按周期执行，到期：
   - 自动创建临时记账记录
   - 推送通知用户确认
   - 用户确认后正式记账
   - 用户可修改后确认

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 待确认 | 模板创建 | 显示创建成功 |
| 暂停 | 点击暂停 | 暂停自动记账 |
| 执行中 | 到期触发 | 创建待确认记录 |
| 已确认 | 用户确认 | 正式记账完成 |

// GET /api/auto-records - 获取自动记账列表
interface GetAutoRecordsResponse {
  code: number;
  data: {
    templates: AutoRecordTemplate[];

interface AutoRecordTemplate {
  id: string;
  type: 'expense' | 'income' | 'transfer';
  categoryId: string;
  amount: string;
  remark: string;
  accountId: string;
  cycle: {

    dayOfMonth?: number;     // 1-31

  };
  startDate: string;
  endDate?: string;
  status: 'active' | 'paused';
  nextRunAt: string;
  createdAt: string;
}

// POST /api/auto-records - 创建模板
// PUT /api/auto-records/:id - 更新模板
// DELETE /api/auto-records/:id - 删除模板

// GET /api/auto-records/pending - 获取待确认记录
interface GetPendingAutoRecordsResponse {
  code: number;
  data: {
    pendingRecords: PendingAutoRecord[];
  };

// POST /api/auto-records/:id/confirm - 确认自动记账
interface ConfirmAutoRecordRequest {

## 边界情况
1. **VIP 到期**：暂停自动记账，通知用户续费
2. **周期日期不存在**：如设置每月31号，2月份跳过
3. **账户被删除**：提示用户更新账户
4. **分类被删除**：提示用户更新分类
5. **金额为0**：不创建记录
6. **历史记录**：模板删除不影响已确认的历史记录
