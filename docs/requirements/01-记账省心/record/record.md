## 🔑 核心设计：账户选择按记账类型差异化

### 记账操作 = 金额 + 分类 + 账户

用户记账不仅仅是记录"花了多少钱、买了什么"，还要明确**钱从哪个账户出、进到哪个账户**。不同的记账类型对应不同的账户选择逻辑：

| 记账场景 | 涉及账户 | 账户选择规则 | 余额变化 |
|----------|----------|-------------|----------|
| **支出** | 1个「支出账户」 | 可选所有资产类 + 负债类账户 | 支出账户余额 ↓ |
| **收入** | 1个「收入账户」 | 仅可选资产类账户（现金类/固定资产类/折旧资产类） | 收入账户余额 ↑ |
| **转账** | 2个：「转出账户」→「转入账户」 | 所有账户可选，转出≠转入 | 转出↓ 转入↑ |
| **还债** | 2个：「还款账户」→「债权账户」 | 转出仅资产类，转入仅负债类 | 还款账户↓ 债权账户↓(负债减少) |

### 为什么收入不能选负债账户？
收入是"钱进来了"，只会进到资产账户（现金、储蓄卡、微信等），不存在"收入进到信用卡"的场景。信用卡还款是「还债」操作，不是「收入」。

### 为什么支出可以选负债账户？
用信用卡、花呗消费时，钱是从负债账户"出去"的（增加负债），所以支出时负债账户也在可选范围。

---

### 账户分类体系

项目已有 4 种账户类型（定义在 `src/types/account.ts`）：

```
📦 资产类（余额 >= 0，绿色显示）
├── 💵 现金类 (cash)          — 支出✅ 收入✅ 转出✅
├── 🏠 固定资产类 (fixed_asset) — 支出✅ 收入✅ 转出✅
└── 📱 折旧资产类 (depreciable_asset) — 支出✅ 收入✅ 转出✅

💳 负债类（余额 <= 0，红色显示）
└── 💳 负债类 (liability)      — 支出✅ 收入❌ 转出✅（还债转入✅）
```

### 账户选择器差异化逻辑

```
记账类型 → 展示的账户列表
─────────────────────────────
支出(expense)
  ┌─ 全部账户（资产+负债）
  │   💵 现金、🏦 储蓄卡、💚 微信、💙 支付宝
  │   💳 信用卡、🌸 花呗
  └─ 默认选中：上次使用的支出账户

收入(income)
  ┌─ 仅资产类账户
  │   💵 现金、🏦 储蓄卡、💚 微信、💙 支付宝
  │   ❌ 不显示负债类
  └─ 默认选中：上次使用的收入账户

转账(transfer)
  ┌─ 转出账户选择器：全部账户
  │   转入账户选择器：全部账户（排除已选转出账户）
  └─ 默认选中：上次使用的转出/转入组合

还债(repayment)
  ┌─ 还款账户选择器：仅资产类账户
  │   债权账户选择器：仅负债类账户
  └─ 默认选中：上次使用的还款/债权组合
```

## 核心设计原则
- ✅ **保持简单**：90%的记账场景（支出/收入）体验不变，只需选一个账户
- ✅ **渐进式增强**：只增加账户选择，不增加复杂度
- ✅ **智能默认**：记住上次使用的账户，下次默认选中
- ✅ **差异化展示**：根据记账类型智能过滤可选账户，减少用户选择困惑

## 用户故事

### 核心记账
作为用户，我希望能快速记录一笔支出或收入，选择分类，输入金额，添加备注，选择日期，然后完成记账。

### 支出时选择账户
作为用户，我记一笔支出时，希望选择钱从哪个账户出去（现金/储蓄卡/微信/信用卡等），这样我能追踪各账户的余额变化。

### 收入时选择账户
作为用户，我记一笔收入时，希望选择钱进到哪个账户（现金/储蓄卡等），**不需要看到信用卡等负债账户**，因为收入不会进到信用卡里。

### 转账
作为用户，我有时候需要记录从储蓄卡转钱到微信这种操作，希望能方便地选择转出和转入两个账户。

### 还债（还信用卡/还花呗/还房贷）
作为用户，我用储蓄卡还信用卡后，希望记录这笔还款：**从储蓄卡（资产账户）转出，信用卡负债（负债账户）减少**。这是一个特殊的转账场景。

### 交互流程

#### 普通支出
1. 用户进入记账页，默认显示支出tab
2. 默认选中上次使用的「支出账户」（可点击切换，展示全部账户）
3. 选择支出分类
4. 弹出金额输入表单（显示选中的支出账户）
5. 输入金额
6. 可选：填写备注、选择日期
7. 点击「完成」提交
8. 该支出账户余额减少
9. 显示「记账成功」，跳转明细页

#### 普通收入
1. 切换到收入tab
2. 默认选中上次使用的「收入账户」（可点击切换，**仅展示资产类账户**）
3. 选择收入分类
4. 弹出金额输入表单
5. 输入金额
6. 可选：填写备注、选择日期
7. 点击「完成」提交
8. 该收入账户余额增加
9. 显示「记账成功」，跳转明细页

#### 转账
1. 支出tab下选择「转账」分类
2. 弹出转账表单：
   - 「转出账户」选择器：全部账户可选
   - 「转入账户」选择器：全部账户可选（排除已选转出账户）
3. 输入金额
4. 可选：填写备注、选择日期
5. 点击「确认转账」提交
6. 转出账户余额减少，转入账户余额增加
7. 显示「转账成功」，跳转明细页

#### 还债
1. 支出tab下选择「还债」分类
2. 弹出还债表单：
   - 「还款账户」选择器：**仅资产类账户可选**
   - 「债权账户」选择器：**仅负债类账户可选**
3. 输入金额
4. 可选：填写备注、选择日期
5. 点击「确认还款」提交
6. 还款账户（资产）余额减少，债权账户（负债）余额减少（欠款变少）
7. 显示「还款成功」，跳转明细页

```typescript
interface CreateRecordRequest {

}
```

### 账户过滤规则（前端实现）
```typescript
// 根据记账类型过滤可选账户
const getFilteredAccounts = (
  allAccounts: Account[],
  transactionType: 'expense' | 'income' | 'transfer' | 'repayment',
  role: 'from' | 'to'  // from=转出/支出方, to=转入/收入方
): Account[] => {
  switch (transactionType) {
    case 'expense':
      // 支出：全部账户可选
      return allAccounts.filter(a => !a.isDeleted && a.isVisible)
    
    case 'income':
      // 收入：仅资产类（现金/固定资产/折旧资产）
      return allAccounts.filter(a => 
        !a.isDeleted && a.isVisible && a.type !== 'liability'
      )
    
    case 'transfer':
      // 转账：转出、转入都是全部账户可选
      return allAccounts.filter(a => !a.isDeleted && a.isVisible)
    
    case 'repayment':
      if (role === 'from') {
        // 还债-还款方：仅资产类
        return allAccounts.filter(a => 
          !a.isDeleted && a.isVisible && a.type !== 'liability'
        )
      } else {
        // 还债-债权方：仅负债类
        return allAccounts.filter(a => 
          !a.isDeleted && a.isVisible && a.type === 'liability'
        )
      }
  }
}
```

## 边界情况

1. **金额为空**：提示「请输入金额」
2. **未选分类**：提示「请选择分类」
3. **重复提交**：提交中禁用按钮
4. **网络错误**：提示「网络错误」
5. **余额不足**：支出或转账时余额不够，仍允许记账（余额可以为负）
6. **同账户转账**：转出和转入账户相同，提示并禁止提交
7. **收入选负债账户**：前端过滤，收入时根本不显示负债账户
8. **还债但无负债账户**：没有负债账户时，「还债」分类不显示债权账户选择器，提示用户先创建
9. **无账户**：用户注册后自动创建默认账户，不会出现无账户情况
10. **账户被删除/隐藏**：只显示 isDeleted=false 且 isVisible=true 的账户
