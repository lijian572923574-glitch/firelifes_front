## 功能概述

### 已实现功能
- **支出/收入切换**：通过tab快速切换记账类型
- **分类选择**：网格布局展示分类，分组显示，iconfont图标
- **金额输入**：大数字键盘，支持简单加减运算
- **日期选择**：支持选择记账日期，默认今天
- **备注功能**：可添加备注信息
- **快速提交**：记账成功后自动跳转明细页

### 待实现功能（本次新增）
- **账户选择**：在支出/收入时选择使用的账户
- **转账分类**：在分类中增加「转账」分类（默认分类，不可删除）
- **默认账户**：用户注册后自动创建默认账户，无需手动添加

## 核心设计原则
- ✅ **保持简单**：90%的记账场景（支出/收入）体验不变
- ✅ **渐进式增强**：只增加账户选择，不增加复杂度
- ✅ **智能默认**：记住上次使用的账户，下次默认选中

## 用户故事

### 核心记账
作为用户，我希望能快速记录一笔支出或收入，选择分类，输入金额，添加备注，选择日期，然后完成记账。

### 账户选择
作为用户，我希望记账时能选择使用的账户，这样我能清楚看到各账户的余额变化。

### 转账
作为用户，我有时候需要记录从储蓄卡转钱到微信这种操作，希望能方便地记录转账。

- 显示收入分类

```
┌────────────────────────────────────┐
```

**转账逻辑：**
- 转出账户余额减少
- 转入账户余额增加
- 不计入收支统计
- 明细中用蓝色显示

### 交互流程

#### 普通支出/收入
1. 用户进入记账页，默认显示支出tab
2. 默认选中上次使用的账户（可点击切换）
3. 选择分类（可切换支出/收入）
4. 弹出金额输入表单（显示选中的账户）
5. 输入金额（支持简单加减：如20+30）
6. 可选：点击填写备注
7. 可选：点击日期按钮选择日期
8. 点击「完成」提交
9. 显示「记账成功」提示
10. 1秒后自动跳转明细页
11. 更新对应账户余额

#### 转账
1. 用户进入记账页，支出tab
2. 选择「转账」分类
3. 弹出转账表单
4. 默认选中上次的转出/转入组合（可点击切换）
5. 输入金额
6. 可选：点击填写备注
7. 可选：点击日期按钮选择日期
8. 点击「确认转账」提交
9. 显示「转账成功」提示
10. 1秒后自动跳转明细页
11. 更新转出/转入账户余额

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 初始 | 页面加载 | 显示支出tab，清空表单，默认选中上次账户 |
| 切换tab | 点击支出/收入 | 清空已选分类，重新加载分类，保持账户选择 |
| 选择分类 | 点击分类 | 弹出对应表单（普通或转账） |
| 输入中 | 点击数字/运算符 | 更新金额显示 |
| 提交中 | 点击完成 | 显示loading，防止重复提交 |
| 成功 | 接口返回成功 | 显示成功提示，跳转明细页，更新账户余额 |

## 数据结构

### 记账记录提交
```typescript
interface CreateRecordRequest {
  typeId: number        // 分类ID
  type: 'expense' | 'income' | 'transfer'  // 记账类型
  amount: number        // 金额（支出为负，收入为正，转账为正）
  accountId: number     // 支出/收入时：使用的账户；转账时：转出账户
  toAccountId?: number  // 转账时：转入账户
  remark?: string       // 备注
  date: string          // 日期（YYYY-MM-DD）
}
```

## 边界情况

1. **金额为空**：提示「请输入金额」
2. **未选分类**：提示「请选择分类」（表单不会弹出，因为必须先选分类）
3. **未选账户**：有默认账户，不会出现此情况
4. **余额不足**：支出或转账时，如果账户余额不足，提示用户
5. **同账户转账**：转账时转出和转入账户相同，提示并禁止提交
6. **新用户无账户**：用户注册后自动创建默认账户，不会出现无账户情况
7. **账户被删除**：选择账户时只显示未删除的账户
8. **重复提交**：提交中禁用按钮，防止重复提交
9. **网络错误**：提示「网络错误」
