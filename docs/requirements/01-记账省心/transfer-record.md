## 功能概述
记账采用**单页设计**，所有流水在一个页面展示，用颜色区分类型。支持：普通支出、收入、转账、还款、借钱。核心逻辑：
- 🔴 红色 = 支出（真实消费，计入支出统计）
- 🟢 绿色 = 收入（真实赚钱，计入收入统计）
- 🔵 蓝色 = 转账/还款/借钱（资金转移，特殊处理）

> **关键逻辑：还款分两类处理**
> - 信用卡/花呗还款：蓝色转账，**不计入**支出统计（避免重复记账）
> - 房贷/车贷还款：蓝色转账，**同时计入**支出统计（符合用户习惯）

## 用户故事
作为用户，我经常需要把钱从储蓄卡转到微信方便消费，或者从微信还到信用卡还款。我希望能把这些转账操作也记录下来，这样我能清楚看到各账户的余额变化，同时支出统计也不会因为还信用卡而虚高。

### 核心交互流程

#### 1. 普通支出/收入（90%场景）
1. 直接输金额
2. 点分类图标（如"餐饮"）
3. 账户用默认的（上次使用的），不用改
4. 完成！自动记录到对应账户

#### 2. 转账模式（微信→支付宝等）
1. 点击「🔄 转账模式」
2. 显示：`从 [💚 微信] 转 ¥____ 到 [💙 支付宝]`
3. 输金额，点确认
4. 系统自动：两个账户余额更新，**不计入**收支统计

#### 3. 还信用卡/花呗
1. 点击「💳 还信用卡」
2. 选择还款账户（银行卡/微信）和负债账户（信用卡/花呗）
3. 输金额，点确认
4. 系统自动：
   - 两个账户余额更新（资产↓，负债↓）
   - **不计入**支出统计（避免消费时重复记账）

#### 4. 还房贷/车贷（特殊处理）
1. 跟转账操作一样
2. 系统自动识别这是"房贷/车贷"账户
3. 系统自动：
   - 两个账户余额更新（资产↓，负债↓）
   - **同时计入**支出统计（"住房"/"交通"分类）

### 账户类型与自动处理规则

| 账户类型 | 还款时是否计支出 | 说明 |
|---------|----------------|------|
| 信用卡/花呗/白条 | ❌ 不计入 | 消费时已记过支出 |
| 房贷/车贷 | ✅ 计入 | 按月还款，用户习惯算支出 |
| 借呗/网贷 | ⚪ 用户可选 | 默认不计，可修改 |
| 私人借款 | ⚪ 用户可选 | 默认不计，可修改 |

### 默认账户（新用户注册自动创建）
```
📦 资产类（4个）
├── 💵 现金
├── 🏦 储蓄卡
├── 💚 微信零钱
└── 💙 支付宝余额

💳 负债类（3个，自带类型标签）
├── 💳 信用卡（还款不计支出）
├── 🌸 花呗（还款不计支出）
└── 🏠 房贷（还款计支出）

💰 收入类（1个，系统内置）
└── 💼 工资收入
```

### 智能推荐与默认值
- 记住上次使用的账户，下次默认选中
- 根据分类智能推荐账户（餐饮→微信，工资→银行卡）
- 转账模式下推荐最近常用的3组转账组合

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 同账户 | 转出=转入 | 提示并禁止提交 |
| 余额不足 | 金额>账户余额 | 提示余额不足 |
| 提交中 | 点击提交 | loading 状态 |
| 成功 | 提交成功 | 显示确认卡片并返回 |

### 关键数据结构
```typescript
// 记账记录类型
type RecordType = 'expense' | 'income' | 'transfer';

// 账户类型（决定还款处理逻辑）
type AccountSubType = 
  | 'cash' | 'debit_card' | 'wechat' | 'alipay'      // 资产类
  | 'credit_card' | 'huabei'                         // 消费贷（还款不计支出）
  | 'mortgage' | 'car_loan'                          // 分期贷（还款计支出）
  | 'loan' | 'borrow_from_friend'                    // 其他借款（用户可选）
  | 'salary' | 'bonus';                               // 收入类

interface Account {
  id: string;
  name: string;
  icon: string;
  type: 'asset' | 'liability' | 'income';
  subType: AccountSubType;
  balance: number;
  // 关键字段：控制还款是否计支出
  countRepaymentAsExpense: boolean;
}
```
