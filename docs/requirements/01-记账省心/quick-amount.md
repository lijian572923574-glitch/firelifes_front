## 功能概述
在金额输入区上方显示常用金额标签，根据该分类的历史金额智能推荐（出现频次最高的3个金额）。用户点击标签可直接填入金额，减少手动输入。

## 用户故事
作为用户，我每次买咖啡都是28元，买早餐都是15元左右。我希望系统能记住我各分类常用的金额，我只需一键选择而非每次都输入数字键盘。

### 交互流程
1. 用户进入记账页，选择分类
2. 系统查询该分类下历史金额
3. 统计出现频次最高的3个金额
4. 显示在金额输入框上方
5. 用户点击金额标签，直接填入输入框
6. 用户可继续编辑或直接提交

### 状态变化
| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 无推荐 | 新分类/无历史 | 显示通用金额 [¥10] [¥50] [¥100] |
| 有推荐 | 有历史数据 | 显示该分类常用金额 |
| 已选择 | 点击标签 | 填入金额，标签高亮 |

## 数据结构

### 新增接口
```typescript
// GET /api/amounts/suggestions - 获取金额推荐
interface GetAmountSuggestionsRequest {
  categoryId: string;
  limit?: number;  // 默认3
}

interface GetAmountSuggestionsResponse {
  code: number;
  data: {
    suggestions: AmountSuggestion[];
  };
}

interface AmountSuggestion {
  amount: string;    // 金额
  count: number;     // 出现次数
  lastUsedAt: string;
}
```

## 边界情况
1. **新用户/无历史**：显示通用推荐 [¥10] [¥50] [¥100]
2. **金额相同频次**：按最后使用时间排序
3. **分类切换**：重新加载推荐
4. **金额输入**：点击标签后，用户可继续编辑
5. **小数金额**：标签显示整数，实际金额支持小数
