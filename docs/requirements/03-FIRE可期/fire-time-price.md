## 功能概述
FIRE 时间换算将每笔支出换算为等价的 FIRE 天数，让用户直观感受消费与财务自由之间的距离。不是让你别花钱，是让你有感知地花钱。

## 用户故事
作为用户，我希望在记账时看到"这杯咖啡 = 0.5 天 FIRE 时间"，这样我可以更理性地消费，知道每一笔钱对实现财务自由的影响，而不是单纯地省钱或内疚。

### 交互流程

1. **记账成功后**
   - 显示记账确认卡片
   - 在金额下方显示 FIRE 时间换算
   - 格式："这笔消费 ≈ X.XX 天 FIRE 时间"

2. **查看明细列表**
   - 每笔支出右侧显示 FIRE 天数小标签
   - 仅显示，不影响布局
   - 可收起/展开

3. **设置开关**
   - 用户可选择开启/关闭此功能
   - 关闭后不在任何地方显示 FIRE 时间
   - 适合觉得焦虑的用户

### 计算公式
```
日均 FIRE 目标支出 = 目标年支出 ÷ 365
                    = (目标月支出 × 12) ÷ 365

FIRE 天数 = 支出金额 ÷ 日均 FIRE 目标支出
          = 支出金额 ÷ (目标月支出 × 12 ÷ 365)
          = 支出金额 × 365 ÷ (目标月支出 × 12)
```

### 状态变化

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 已开启 | 设置中开启 | 显示 FIRE 时间 |
| 已关闭 | 设置中关闭 | 不显示 FIRE 时间 |
| 无 FIRE 目标 | 未设定 FIRE 目标 | 不显示 FIRE 时间 |
| 支出显示 | 支出记录 | 显示 FIRE 天数 |
| 收入隐藏 | 收入记录 | 不显示 FIRE 天数 |

## 数据结构

### 前端计算逻辑
```typescript
// 计算 FIRE 天数
function calculateFireDays(expense: number, fireGoal: FireGoal): number {
  const annualExpense = fireGoal.targetMonthlyExpense * 12;
  const dailyExpense = annualExpense / 365;
  return expense / dailyExpense;
}

// 显示格式化
function formatFireDays(days: number): string {
  if (days < 0.01) return '< 0.01 天';
  if (days < 1) return `${(days * 24).toFixed(0)} 小时`;
  if (days < 7) return `${days.toFixed(1)} 天`;
  if (days < 30) return `${(days / 7).toFixed(1)} 周`;
  return `${(days / 30).toFixed(1)} 月`;
}

// 示例：
// ¥35 咖啡 ≈ 0.5 天
// ¥200 晚餐 ≈ 2.9 天
// ¥5000 旅行 ≈ 71.4 天 ≈ 2.4 月
```

## 边界情况

1. **未设定 FIRE 目标**
   - 不显示 FIRE 时间
   - 不计算

2. **支出金额为0**
   - 显示"0 天"

3. **支出金额极小**
   - 如 ¥0.01
   - 显示"< 0.01 天"

4. **支出金额极大**
   - 如 ¥100,000
   - 显示具体月数/年份
   - 如 "≈ 3.9 月"

5. **用户关闭功能**
   - 完全不显示
   - 减少认知负担

6. **觉得焦虑的用户**
   - 可以关闭
   - 不强制显示

7. **负数金额（退款）**
   - 负数表示减少 FIRE 天数
   - 如退款 ¥35 → "-0.5 天"

8. **目标月支出为0**
   - 不计算
   - 不显示
