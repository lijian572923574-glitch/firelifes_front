## 功能概述

每次记账完成后，不再简单地跳转回明细页，而是展示一张确认卡片，包含当前净资产和距离FIRE目标的差距。让每一次记账都成为一次"距离自由还有多远"的提醒，给用户即时情绪反馈。

## 用户故事
作为用户，每次记完一笔账，我希望看到的不只是"记账成功"，而是我距离FIRE目标还有多远。尤其是大额支出时，看到FIRE距离被拉远，会让我更有动力去控制消费、增加储蓄。

### 交互流程

1. 用户点击「完成」提交记账
2. 显示 loading 遮罩（当前已有）
3. 记账成功 → loading 切换为确认卡片
4. 卡片中异步加载净资产数据（getNetWorth）
5. 用户选择：
   - **继续记**：关闭卡片，清空表单，保持交易类型和账户选择
   - **返回明细**：跳转 detail/index.vue

### 状态变化

| 状态 | 显示 |
|------|------|
| 记账成功（普通支出） | 金额 + 分类名 + 账户余额 + 净资产 + FIRE进度 |
| 记账成功（记入资产） | 增加"已记入折旧资产"标识 |
| 记账成功（收入） | 金额前"+"号，绿色文字 |
| 净资产加载中 | 进度条显示骨架屏动画 |
| 无FIRE目标 | 不显示进度条，显示"设定FIRE目标"引导按钮 |

### 动效
- 确认卡片：scale 0.9→1 + fadeIn，200ms
- 打勾图标：弹性动画，300ms
- FIRE进度条：宽度从0过渡到目标值，600ms easeOut
- 净资产数字：数字滚动动画，800ms

## 数据结构

### 前端状态

```typescript
interface ConfirmCardData {
  amount: number;           // 记账金额
  type: 'income' | 'expense';
  categoryName: string;     // 分类名
  accountName: string;      // 账户名
  accountIcon: string;      // 账户图标
  accountBalance: number;   // 更新后账户余额
  hasAsset: boolean;        // 是否记入了折旧资产
  netWorth: number;         // 净资产（异步获取）
  fireGoal: number;         // FIRE目标（异步获取）
  fireProgress: number;     // 进度百分比
}
```

### 净资产加载策略
```
1. 提交记账 → 成功
2. 显示确认卡片骨架（金额+账户立即展示）
3. 异步请求 getNetWorth
4. 净资产数据返回 → 动画展示
5. 如果 500ms 未返回 → 显示"计算中..."
6. 如果请求失败 → 不显示净资产模块，仅显示记账结果
```

## 边界情况

1. **净资产数据加载失败**：不展示净资产模块，仅展示记账结果
2. **未设定FIRE目标**：展示"设定FIRE目标"引导按钮，不展示进度条
3. **快速连续记账**：点击「继续记」前确保上一次请求完成
4. **账户余额变更**：使用记账成功后的最新余额
5. **离线场景**：不请求净资产，仅展示记账结果
