## 功能概述

从分析页/资产管家的账户列表点击账户卡片，进入该账户的交易明细页，按时间倒序展示该账户关联的所有收支记录。用户可以从"账户"维度而非"月份"维度回溯资金变动，解决"余额变了但不知道为什么"的问题。

## 用户故事

作为用户，我希望点击某个账户就能看到它的所有收支流水，这样当账户余额发生变化时，我可以快速追溯钱的来源和去向，而不需要在按月份浏览的明细页里逐条翻找。

## 与现有功能的关系

### 依赖
- **record-account-linkage.md**：记账时已绑定 accountId，数据基础已具备
- **asset-overview.md**：分析页已有账户余额列表，提供入口

### 区别于 account-detail.md
- `account-detail.md` 是账户的完整管理页（余额变动历史 + 对账 + 编辑 + 删除），偏管理视角
- 本需求是从资产视角快速查看交易流水，轻量、只读，聚焦"这个账户的钱怎么变的"
- 两者共享后端查询接口，但前端页面和交互独立

### 入口位置
- **主入口**：分析页（`pages/analysis/index.vue`）账户余额列表 → 点击账户卡片行
- **后续可扩展**：明细首页 FunctionBar 的"资产管家"入口、记账页账户选择器

### 交互流程

```
分析页                           账户交易明细页
┌─────────────┐                 ┌──────────────────┐
│ 账户余额     │                 │  ← 账户交易明细   │
│ ┌─────────┐ │   点击账户行     │                    │
│ │现金 ¥3210│ │ ─────────────→ │  账户信息+流水列表  │
│ └─────────┘ │                 │                    │
│ ┌─────────┐ │                 └──────────────────┘
│ │储蓄 ¥5k │ │ ─────────────→  同上
│ └─────────┘ │
└─────────────┘
```

1. **进入页面**
   - 从分析页账户列表点击某账户行
   - 携带参数：accountId
   - 加载账户详情 + 该账户全部交易记录

2. **头部账户信息区**
   - 展示账户图标、名称、当前余额
   - 展示本月收支小计（支出/收入）
   - 余额为负时红色显示，正数绿色

3. **交易列表**
   - 按日期分组（今天/昨天/MM月DD日 星期X）
   - 每条记录展示：分类图标 + 分类名 + 金额 + 备注（如有）
   - 支出/转出金额红色带 `-`，收入/转入金额绿色带 `+`
   - 转账记录显示方向：`转出至XX` / `从XX转入`
   - 还债记录显示：`还款至XX`

4. **点击记录**
   - 跳转到编辑记录页（`pages/record/edit-record`），传入 recordId

5. **分页加载**
   - 首次加载 50 条，上拉加载更多
   - 无更多数据时显示"已经到底了"

6. **空态**
   - 无交易记录时显示"暂无交易记录，开始记账吧"

### 状态变化

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| Loading | 页面加载中 | 骨架屏 |
| 正常 | 有交易记录 | 展示账户信息+流水列表 |
| 空态 | 无交易记录 | 空状态引导 |
| 加载更多 | 上拉触底 | 加载下一页 |
| 全部加载 | 无更多数据 | 底部显示"已经到底了" |

### 新增后端接口

```typescript
// GET /record/by-account/:accountId - 按账户查询交易记录
interface GetRecordsByAccountQuery {
  page?: number       // 页码，默认1
  pageSize?: number   // 每页条数，默认50
}

interface GetRecordsByAccountResponse {
  success: boolean
  data: {
    account: {
      id: string
      name: string
      icon: string
      type: AccountType
      balance: number
    }
    monthlySummary: {
      income: number    // 本月收入
      expense: number   // 本月支出
    }
    records: PageResult<RecordItem>
  }
}

interface RecordItem {
  id: number
  type: 'income' | 'expense' | 'transfer' | 'repayment'
  amount: number
  date: string
  remark: string
  categoryId: number
  categoryName: string
  categoryIcon: string
  // 转账/还债时的对方账户
  counterpartAccountName?: string
}
```

### 后端实现要点

1. **查询范围**：`accountId` 匹配 `record.accountId`（转出方）或 `record.toAccountId`（转入方）
2. **转账方向判断**：
   - `record.accountId === 目标accountId` → 转出，金额显示为负
   - `record.toAccountId === 目标accountId` → 转入，金额显示为正
3. **分类信息关联**：通过 typeId 关联 user_category_customizations 表获取分类名和图标
4. **分页排序**：按 date DESC, createdAt DESC

## 边界情况

1. **账户不存在或已删除**
   - 显示空页面，提示"账户不存在或已删除"
   - 提供返回按钮

2. **账户无任何交易记录**
   - 显示账户信息（余额为初始值）
   - 列表区域空态："暂无交易记录，开始记账吧"

3. **跨账户转账**
   - 在转出方显示"转出至[对方账户名]"，金额为负
   - 在转入方显示"从[对方账户名]转入"，金额为正
   - 同一笔转账在两个账户的交易明细中都应出现

4. **还债记录**
   - 在还款账户显示"还款至[债权账户名]"，金额为负
   - 在债权账户显示"收到还款[从还款账户名]"，金额为正

5. **记录量很大**
   - 分页加载，每页 50 条
   - 上拉加载更多，避免一次加载全量

6. **账户余额为0**
   - 正常显示 ¥0.00，颜色为默认绿色（非负债账户）
