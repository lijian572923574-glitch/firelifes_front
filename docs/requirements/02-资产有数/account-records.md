## 功能概述

从分析页账户列表点击账户卡片，进入该账户的交易明细页。参考鲨鱼资产管家的交互：顶部简洁展示账户信息和余额，下方展示该账户的全部交易流水，按日期分组。用户可以从"账户"维度回溯资金变动，解决"余额变了但不知道为什么"的问题。

## 用户故事

作为用户，我希望点击某个账户就能看到它的所有收支流水，这样当账户余额发生变化时，我可以快速追溯钱的来源和去向，而不需要在按月份浏览的明细页里逐条翻找。

### 交互流程

1. **进入页面** — 从分析页账户列表点击某账户行，携带 accountId
2. **导航栏** — 顶部左侧显示"←"返回按钮，点击返回分析页（H5刷新后无历史栈时兜底跳转分析页）；中间显示"账户交易明细"标题
3. **头部账户信息** — 图标、名称、金额同行显示（金额右对齐），下方本月收支小计
4. **交易列表** — 按日期分组，每条：分类图标+分类名（转账显示方向）+金额（支出红/收入绿）
5. **点击记录** — 跳转编辑记录页
6. **上拉加载** — 分页50条，触底加载更多
7. **空态** — "暂无交易记录"

### 与鲨鱼资产管家的差异

| 对比项 | 鲨鱼资产管家 | 本方案 |
|--------|-------------|--------|
| 头部 | 账户名+余额+"更新余额"按钮 | 图标+名称同行 + 余额 + 本月收支 |
| 列表内容 | 余额调整记录 | 实际交易记录（有分类、备注） |
| 列表分组 | 无分组，纯时间线 | 按日期分组，每天显示收支小计 |
| 交互 | 更新余额 | 点击记录可跳转编辑 |

本方案比鲨鱼更优：因为记账时已联动账户余额，交易记录本身就反映了余额变动，不需要额外的"余额调整"功能。

## 边界情况

1. 账户不存在 — 提示"账户不存在"，提供返回按钮
2. 无交易记录 — 显示账户信息 + 空态
3. 转账记录 — 转出方显示"转出至XX"，转入方显示"从XX转入"
4. 还债记录 — 还款方显示"还款至XX"，债权方显示"收到还款"
5. 记录量大 — 分页加载，每页50条
