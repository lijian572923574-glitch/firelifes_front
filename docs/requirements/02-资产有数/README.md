## 模块概述

资产有数的核心理念是"不用手动对账，记账自动更新资产，一目了然"。账户从"记账标签"升级为"一等公民"，成为个人财务管理的核心实体。用户可以创建、维护多种类型的账户，每种账户类型有不同的数据结构和管理方式。

核心新增：**净资产计算引擎**作为三支柱的桥梁，连接"记账省心"和"FIRE可期"。

## 模块数据流

```
记账记录
├── 选择账户 ───→ 更新账户余额
├── 记入资产 ───→ 创建折旧资产
└── 提交成功

净资产计算
├── Σ(现金账户余额)
├── + Σ(折旧资产当前价值)
├── + Σ(固定资产估值)
├── - Σ(负债账户余额)
└── = 净资产 → 推送 FIRE可期模块
```

## 净资产公式

```
净资产 = Σ(所有账户余额，含负债负值) + Σ(折旧资产.当前价值)
```

核心原则：
- 支出记账时，现金账户余额减少（计入资产形态转换，净资产不变）
- 随时间折旧，折旧资产价值下降（净资产缓慢减少，反映价值损耗）
- 负债类余额为负值，减少净资产

## 账户类型说明

### 资产类
| 类型 | 说明 | 价值计算 |
|------|------|----------|
| 现金类 | 余额即价值 | 直接使用账户余额 |
| 折旧资产 | 自动折旧 | 购买价 - 累计折旧 |
| 固定资产 | 估值+负债 | 用户手动估值 |

### 负债类
| 类型 | 说明 | 记录方式 |
|------|------|----------|
| 信用卡 | 消费后还款 | 负余额显示 |
| 贷款 | 分期/房贷等 | 显示剩余本金 |

## 与其他模块的关联

### 依赖关系
- **记账省心**：依赖记账数据更新账户余额、创建折旧资产

### 被依赖关系
- **FIRE可期**：依赖净资产数据计算FIRE进度，是FIRE可期的唯一数据输入
