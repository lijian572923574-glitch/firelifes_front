## 一、功能概述

### 功能目标
子分类新增/编辑弹窗是用户在子分类列表页通过点击"+"按钮（新增）或左滑编辑按钮（编辑）时弹出的底部 Sheet，用户在此选择图标、输入名称，完成子分类的创建或更新。弹窗需要展示所属大类和收支类型信息（只读不可改，因为子分类的类型跟随大类）。

### 核心变更（v1.0 相较于原型设计）

| 变更项 | 旧设计 | 新设计 |
|--------|--------|--------|
| **所属大类** | 不展示 | **展示**：灰色底标签，只读不可改 |
| **收支类型** | 新增时可选 | **展示**：灰色底标签，只读不可改（跟随大类） |
| **信息来源** | 无 | 从父级路由参数 `groupName` 和 `groupType` 传入 |

### 入口
1. **新增**：子分类列表页右上角"+"按钮
2. **编辑**：子分类卡片左滑后点击"编辑"按钮

## 二、用户故事

**作为** 记账用户，
**我想要** 在新增/编辑子分类时看到所属大类和收支类型的只读展示，
**以便于** 明确知道该子分类与哪个大类关联、属于支出还是收入，并且不会误修改这些跟随大类的信息。

**验收标准：**
- 弹窗打开时，所属大类名称正确显示（如"餐饮"）
- 弹窗打开时，收支类型正确显示（如"💸 支出"），与所属大类一致
- 这两个字段无法交互/修改，仅展示
- 弹窗底部有取消和确定按钮
- 点击取消或遮罩关闭弹窗，不提交
- 点击确定提交：新增走 create 接口，编辑走 update 接口

### 3.2 只读字段设计（v1.0 核心变更）

| 字段 | 数据来源 | 展示样式 | 可否修改 |
|------|---------|---------|---------|
| **所属大类** | 路由参数 `groupName`（如"餐饮"） | 灰色底标签（`var(--color-bg-page)` 底色 + `var(--color-text-secondary)` 文字），圆角 6rpx，字号 14px | ❌ 不可修改 |
| **收支类型** | 路由参数 `groupType`（如 `expense`） | 灰色底标签，`expense` → "💸 支出"，`income` → "💰 收入" | ❌ 不可修改 |

**设计意图**：子分类的类型必须跟随所属大类。例如大类"餐饮"是支出类，其下所有子分类也是支出类。因此新增/编辑时不需要也不应该让用户修改类型。通过只读展示，用户可以：
- 确认新增的分类将归属于哪个大类
- 确认该分类的收支类型与预期一致

### 3.3 图标选择器

| 交互元素 | 规则 |
|---------|------|
| **展示形式** | 横向滚动列表，每行显示约 5~6 个图标 |
| **图标集合** | 从后端接口获取或使用预设 emoji 集合 |
| **选中态** | 1px `var(--color-primary)` 边框 + 渐变浅主色背景 |
| **默认选中** | 新增模式：选中第一个图标；编辑模式：选中当前图标 |
| **交互** | 点击单个图标切换选中，支持横向滚动 |

### 3.4 名称输入

| 规则 | 说明 |
|------|------|
| 最大长度 | 6 个中文字符 |
| 字数计数器 | 输入框右下方显示 `当前字数/6` |
| 占位符 | "请输入分类名称"（新增）/ 预填当前值（编辑） |
| 校验 | 不能为空、不能重复（同一大类下） |

### 3.5 底部按钮

| 按钮 | 样式 | 行为 |
|------|------|------|
| 取消 | `var(--color-text-secondary)` 文字，`var(--color-bg-card)` 底，圆角 10rpx | 关闭弹窗，不提交 |
| 确定 | `var(--color-text-inverse)` 文字，`var(--color-primary)` 渐变底，圆角 10rpx | 提交数据，成功后关闭弹窗 + toast + 刷新列表 |

### 3.6 弹窗模式差异

| 字段 | 新增模式 | 编辑模式 |
|------|---------|---------|
| 标题 | "新增子分类" | "编辑子分类" |
| 图标 | 默认选中第一个 | 预填当前图标 |
| 名称 | 空 | 预填当前名称 |
| 所属大类 | 灰色标签（只读） | 灰色标签（只读） |
| 收支类型 | 灰色标签（只读） | 灰色标签（只读） |

## 六、数据结构

### 请求/响应参数

```typescript
// 新增子分类
interface CreateCategoryRequest {
  name: string;         // 分类名称
  groupId: number;      // 所属大类 ID（从路由参数获取）
  iconId: number;       // 图标 ID
  type: 'income' | 'expense'; // 跟随大类类型
}

// 编辑子分类
interface UpdateCategoryRequest {
  id: number;           // 子分类 ID
  name: string;         // 分类名称
  iconId: number;       // 图标 ID
  // 注意：groupId 和 type 不可修改，不需要传
}

// 响应
interface CategoryResponse {
  code: number;
  message: string;
  data: {
    id: number;
  };
}
```

### 弹窗本地状态

```typescript
interface EditFormState {
  mode: 'create' | 'edit';
  groupId: number;       // 所属大类 ID
  groupName: string;     // 所属大类名称（只读展示）
  groupType: 'income' | 'expense'; // 所属大类类型（只读展示）
  iconId: number;
  name: string;
  categoryId?: number;   // 编辑模式下的子分类 ID
}
```

## 八、校验规则

| 规则 | 触发条件 | 提示文案 |
|------|---------|---------|
| 名称为空 | 提交时 | "请输入分类名称" |
| 名称超长 | 输入时 | 限制输入，最多 6 个中文字 |
| 名称重复 | 提交时 | "该分类名称已存在" |
| 图标未选 | N/A | 默认选中第一个，无需校验 |

## 九、异常处理

| 场景 | 处理方式 |
|------|---------|
| 网络错误 | toast 提示"网络异常，请重试"，弹窗不关闭 |
| 服务端错误 | toast 提示服务端返回的 message |
| 名称重复 | toast 提示"该分类名称已存在" |
| 图标列表加载失败 | toast 提示"图标加载失败"，显示空图标列表 |
