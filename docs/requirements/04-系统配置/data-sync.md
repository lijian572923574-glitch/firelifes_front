## 功能概述

数据同步模块确保用户数据在本地和服务端保持一致，支持离线记账和联网后自动同步，提供数据导出和恢复功能，保障数据安全。

## 用户故事

作为用户，我希望即使在地铁等网络不好的地方也能记账，联网后数据自动同步到云端，同时可以导出数据备份或在换手机时恢复数据，确保我的财务记录永不丢失。

### 交互流程

#### 1. 自动同步
- 开启自动同步后，在以下情况触发同步：
  - 应用启动且联网
  - 记账提交成功后
  - 从后台切换到前台
  - 网络恢复连接
- 同步期间显示同步状态指示器
- 同步完成后更新"上次同步时间"

#### 2. 手动同步
- 用户点击"立即同步"
- 显示同步进度
- 完成后显示成功提示

#### 3. 离线记账
- 关闭网络或网络不佳时
- 记账数据保存到本地队列
- 显示"离线缓存：X 条记录待同步"
- 联网后自动触发同步

#### 4. 冲突处理
- 当本地和服务端数据冲突时
- 以服务端数据为准
- 本地记录标记为冲突
- 用户可在明细中查看冲突记录

#### 5. 数据导出
- 用户选择导出时间范围和格式
- 点击"开始导出"
- 生成文件后打开分享面板
- 可分享到微信、邮箱或保存到本地

#### 6. 数据恢复
- 用户点击"从服务端恢复数据"
- 确认弹窗提示风险
- 从服务端拉取最新数据
- 覆盖本地数据

### 状态变化

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| 已同步 | 同步完成 | 显示"已同步" |
| 同步中 | 正在同步 | 显示"同步中..." |
| 同步失败 | 同步出错 | 显示"同步失败，点击重试" |
| 离线模式 | 网络断开 | 显示"离线模式" |
| 待同步 | 离线记账 | 显示待同步数量 |
| 冲突 | 数据冲突 | 标记冲突记录 |

## 数据结构

### 同步相关接口
```typescript
// 同步请求
interface SyncRequest {
  lastSyncTime: string;                    // 上次同步时间
  changes: {
    records: RecordChange[];              // 记账记录变更
    accounts: AccountChange[];             // 账户变更
    categories: CategoryChange[];         // 分类变更
  };
}

interface RecordChange {
  type: 'create' | 'update' | 'delete';
  record: Record;
  changedAt: string;
}

interface AccountChange {
  type: 'create' | 'update' | 'delete';
  account: Account;
  changedAt: string;
}

// 同步响应
interface SyncResponse {
  code: number;
  data: {
    serverChanges: {
      records: Record[];
      accounts: Account[];
      categories: Category[];
    };
    conflictRecords: ConflictRecord[];     // 冲突记录
    syncTime: string;                      // 同步时间
  };
}

interface ConflictRecord {
  localRecord: Record;
  serverRecord: Record;
  resolvedRecord: Record;                   // 解决后的记录
  conflictType: 'modified' | 'deleted';
}

// 导出接口
// POST /api/export
interface ExportRequest {
  startDate?: string;
  endDate?: string;
  format: 'csv' | 'excel';
  includeRecords: boolean;
  includeAccounts: boolean;
  includeCategories: boolean;
  includeAssets: boolean;
  includeFireGoal: boolean;
}

interface ExportResponse {
  code: number;
  data: {
    fileUrl: string;                       // 导出文件下载地址
    fileName: string;
  };
}

// 恢复接口
// POST /api/recover
interface RecoverRequest {
  confirm: boolean;                        // 确认恢复
}

interface RecoverResponse {
  code: number;
  data: {
    recovered: {
      records: number;
      accounts: number;
      categories: number;
    };
  };
}
```

## 边界情况

1. **网络中断**
   - 自动切换离线模式
   - 数据保存到本地队列
   - 联网后自动同步

2. **同步失败**
   - 自动重试 3 次
   - 重试间隔递增
   - 仍失败则显示错误提示

3. **数据冲突**
   - 以服务端数据为准
   - 标记冲突记录
   - 用户可在明细查看

4. **导出文件过大**
   - 限制单次导出数量
   - 超过限制时分批导出

5. **恢复数据**
   - 确认弹窗提示
   - 覆盖本地数据
   - 不可撤销

6. **多设备同时修改**
   - 服务端时间戳判断
   - 最新修改优先

7. **存储空间不足**
   - 提示用户清理
   - 优先清理过期缓存

8. **同步过程中退出应用**
   - 保存同步进度
   - 下次启动继续

## 安全规范

1. **数据加密**
   - 传输使用 HTTPS
   - 敏感数据加密存储

2. **隐私保护**
   - 导出文件不包含敏感配置
   - 用户数据隔离存储

3. **备份策略**
   - 服务端保留 30 天数据
   - 用户可导出完整备份
