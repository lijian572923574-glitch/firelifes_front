## 功能概述

负债类账户（贷款模式）记录房贷、车贷、消费贷、私人借款等欠款，支持配置贷款参数，实现自动计算每期还款额、自动拆分本金利息、自动记账。核心目标是让用户一次性录入贷款信息，之后全程自动处理。

> **v2.0 重要变更**：信用卡已独立为第六大类账户 `credit_card`，详见 [信用卡账户规则](./credit-card-rules.md)。负债类账户现在专指贷款类负债。

## 用户故事

作为用户，我希望能为负债类账户（贷款）配置贷款参数（利率、期数、还款方式等），让系统自动计算每月还款额并自动拆分本金和利息，这样我不用手动计算就能精准管理房贷等负债。

## 核心设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                       设计原则总览                             │
├─────────────────────────────────────────────────────────────┤
│  1. 用户只维护静态基础信息，不维护每期还款金额                 │
│  2. 每期实际还款额、本息拆分，全靠公式自动运算                  │
│  3. 完美兼容：固定月供 / 递减月供 / 先息后本 / 灵活还款       │
│  4. 房产固定资产账户保持不动，只动现金流 + 负债                 │
│  5. 灵活还款按实际天数计息，用户自由输入还款金额               │
└─────────────────────────────────────────────────────────────┘
```

## 二、四种还款方式计算规则

### 基础公式定义

```
月利率 i = 年利率 ÷ 12 ÷ 100
P = 当期剩余本金
P总 = 原始贷款总本金
N = 总期数
n = 剩余期数
```

### 1. 等额本息（最常见房贷方式）

```
┌─────────────────────────────────────────────────┐
│  特点：每月还款额固定不变，前期利息多、后期本金多  │
├─────────────────────────────────────────────────┤
│  固定月供 M = P × [i(1+i)ⁿ] ÷ [(1+i)ⁿ - 1]      │
│  当月利息 = P × i                                │
│  当月还本 = M - 当月利息                         │
│  本月现金流出 = M（固定不变）                     │
└─────────────────────────────────────────────────┘
```

### 2. 等额本金

```
┌─────────────────────────────────────────────────┐
│  特点：月供逐月递减，每月还本金固定，利息逐月减少  │
├─────────────────────────────────────────────────┤
│  每月固定还本额 = P总 ÷ N                        │
│  当月利息 = 当期剩余本金 × i                     │
│  本月现金流出 = 固定本金 + 当月利息               │
│  （每期流出金额自动递减，无需人工设置）           │
└─────────────────────────────────────────────────┘
```

### 3. 先息后本

```
┌─────────────────────────────────────────────────┐
│  特点：前期只还利息，末期一次性还清本金           │
├─────────────────────────────────────────────────┤
│  正常还款期：本月流出 = 当期剩余本金 × i          │
│  最后一期结清：流出 = 剩余全部本金 + 当期利息     │
└─────────────────────────────────────────────────┘
```

### 4. 灵活还款（亲友借款模式）—— v2.0 完整落地

```
┌─────────────────────────────────────────────────────────────┐
│  特点：无固定月供、无固定还款日、想还多少还多少、随时可还    │
│  适用场景：借亲戚朋友的钱、私人之间无固定期限借款             │
├─────────────────────────────────────────────────────────────┤
│  核心规则：                                                 │
│  1. 利率可选：无息（年利率填0） / 约定年利率                │
│  2. 不自动触发还款日提醒，用户手动操作还款                   │
│  3. 每次还款金额用户自由输入（≥0）                           │
│  4. 系统自动拆分：有息时先扣息、后还本                       │
│  5. 无总期数、无剩余期数限制                                │
│  6. 可随时一次性全额结清                                    │
├─────────────────────────────────────────────────────────────┤
│  若约定有息（annualInterestRate > 0）：                      │
│    应还利息 = 剩余本金 × 年利率 ÷ 100 × 距上次还款天数 ÷ 365 │
│    本次还本金 = 本次还款总额 - 应还利息                      │
│    （利息滚入下一期，不产生复利）                            │
│                                                             │
│  若约定无息（annualInterestRate = 0）：                      │
│    全部还款额直接冲减本金                                    │
│    利息 = 0                                                 │
└─────────────────────────────────────────────────────────────┘
```

#### 灵活还款交互流程

```
用户进入负债账户详情页
    ↓
点击 [去还款] 按钮
    ↓
进入还款操作页：展示当前剩余本金、上次还款日期、距上次还款天数
    ↓
输入本次还款金额（任意金额 ≥0）
    ↓
系统实时计算并展示：
┌─────────────────────────────────┐
│  本次还款：¥ X,XXX              │
│  其中利息：¥ XXX（按实际天数）  │
│  其中本金：¥ X,XXX             │
│  还款后剩余本金：¥ XX,XXX      │
│                                 │
│  ⚠️ 若还款金额不足利息，        │
│     差额将滚入下期（不产生复利）│
└─────────────────────────────────┘
    ↓
用户确认 → 生成记账记录
    ↓
┌─────────────────────────────────┐
│  1. 流动资产减少 = 还款总额     │
│  2. 负债账户减少 = 本金部分     │
│  3. 利息部分计入「利息支出」    │
└─────────────────────────────────┘
    ↓
更新：剩余本金、lastRepaymentDate
```

#### 灵活还款边界情况处理

| 场景 | 处理规则 |
|------|---------|
| 还款金额 = 0 | 不允许提交，提示"请输入有效还款金额" |
| 还款金额 < 应还利息 | 全部冲抵利息，剩余利息滚入下期；本金不变 |
| 还款金额 = 剩余本金 + 利息 | 全额结清，负债余额变为 0 |
| 还款金额 > 剩余本金 + 利息 | 不允许，提示"还款金额不能超过剩余本金+利息" |
| 首次还款（无上次还款日） | 从账户创建日开始计息 |
| 年利率 = 0（无息） | 全部还款冲本金，利息始终为 0 |

## 四、完整账务分录（以房贷为例）

**场景**：月供 8000 元，其中本金 3000 元、利息 5000 元

| 账务项 | 金额变动 | 说明 |
|--------|---------|------|
| 流动资产（银行卡） | -8000 | 实际现金总流出 |
| 负债（住房贷款） | -3000 | 归还本金，负债减少 |
| 支出（房贷利息） | +5000 | 利息计入生活开销 |
| 固定资产（房产） | 0 | 房产账户全程不动 |

### 对报表的影响

| 报表类型 | 影响 | 数据表现 |
|---------|------|---------|
| 流水视角 | 总支出 8000 | 全部算"往外花钱" |
| 资产盘点视角 | 总资产不变 3000 + 减少 5000 | 本金还债调结构，利息纯消费 |
| 月度支出报表 | 计 5000 元支出 | 只有利息计入生活成本 |
| 资产负债表 | 资产 -8000、负债 -3000 | 净资产净减少 5000（利息部分） |

## 五、数据结构设计

### 前端类型定义

```typescript
// types/account.ts

// 还款方式（扩展含 flexible）
export type RepaymentMethod = 
  | 'equal_principal_interest'  // 等额本息
  | 'equal_principal'            // 等额本金
  | 'interest_first'             // 先息后本
  | 'flexible';                  // 灵活还款（亲友借款）

// Account 接口中的贷款相关字段
export interface Account {
  // ... 原有字段
  autoRepaymentEnabled: boolean;               // 是否启用自动记账
  originalPrincipal?: number;                  // 原始贷款总本金
  annualInterestRate?: number;                  // 贷款年利率
  repaymentMethod?: RepaymentMethod;            // 还款方式
  totalMonths?: number;                         // 总还款期数
  remainingMonths?: number;                     // 剩余还款期数
  repaymentDay?: number;                        // 每月还款日 1-28
  linkedAssetAccountId?: string;                // 关联资产账户ID
  lastRepaymentDate?: string;                   // 上次还款日期（灵活还款用）
}

// 还款计算结果
export interface RepaymentCalculation {
  totalPayment: number;         // 本期应还总额
  principal: number;            // 本期归还本金
  interest: number;             // 本期利息
  remainingPrincipal: number;   // 还款后剩余本金
  remainingMonths: number;       // 还款后剩余期数
  daysSinceLastPayment?: number; // 距上次还款天数（灵活还款专用）
}
```

## 六、API 接口设计

### 1. 计算当期还款额

```
GET /api/accounts/:id/calculate-repayment
```

**响应**：
```json
{
  "success": true,
  "data": {
    "totalPayment": 8000.00,
    "principal": 3000.00,
    "interest": 5000.00,
    "repaymentMethod": "equal_principal_interest",
    "remainingPrincipal": 997000.00,
    "nextPaymentDate": "2026-06-21"
  }
}
```

### 2. 灵活还款计算接口

```
POST /api/accounts/:id/calc-flexible-repayment
```

**请求体**：
```json
{
  "amount": 5000
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "totalPayment": 5000.00,
    "principal": 4800.00,
    "interest": 200.00,
    "daysSinceLastPayment": 30,
    "remainingPrincipal": 95200.00,
    "isFullSettlement": false
  }
}
```

### 3. 生成还款计划预览

```
GET /api/accounts/:id/repayment-schedule?months=12
```

（仅适用前三种还款方式，灵活还款返回空数组）

### 4. 执行还款记账

```
POST /api/accounts/:id/execute-repayment
```

**请求体**（灵活还款时必传 amount）：
```json
{
  "amount": 5000
}
```

## 七、定时任务设计

### 每日检查任务（仅针对前三种还款方式）

```
每日凌晨 01:00 执行
    ↓
遍历所有 autoRepaymentEnabled = true
      AND repaymentMethod IS NOT NULL
      AND repaymentMethod != 'flexible' 的负债账户
    ↓
检查今天是否是还款日（today.day === repaymentDay）
    ↓
如是 → 自动计算当期还款额并生成记账记录
    ↓
更新账户：balance（余额）、remainingMonths、lastRepaymentDate
    ↓
发送通知：本月贷款已自动记账，本金xxx，利息xxx
```

### 灵活还款

- 不参与定时任务检查
- 不自动提醒还款日
- 用户主动进入账户详情页手动操作还款
- 还款金额完全由用户决定，系统只做利息计算和拆分
