## 功能概述

信用卡作为独立的第六大类账户，不同于贷款类负债。信用卡基于账单周期运转，核心概念是**账单日 → 还款日 → 免息期**。用户持续消费，每月账单日汇总出账，还款日前全额还清则享受免息期；选择最低还款或部分还款则按日计息。

> **为什么独立为第六大类？** 信用卡与贷款（负债类）有本质差异：没有固定本金、没有固定期数、利率仅在逾期时生效（免息期内为0）、还款金额由用户决定而非自动计算。

## 用户故事

1. 作为用户，我希望能创建信用卡账户，设置账单日和还款日，系统自动提醒我每月还款。
2. 作为用户，我希望信用卡消费自动记录到信用卡账户，每月账单日能看到本期应还金额。
3. 作为用户，在还款时能选择全额还款（免息）、最低还款（按日计息）或自定义金额。
4. 作为用户，我想看到每张信用卡的额度使用率，合理规划消费。

## 二、信用卡账户核心字段

### 字段配置表

当账户类型选择「信用卡类」（`type === 'credit_card'`）时，表单展示 `CreditCardForm` 独立组件，包含以下字段：

| 字段 | 必填 | 类型 | 默认值 | 说明 |
|------|------|------|--------|------|
| **账单日** | ✅ | 1-28 | - | 每月出账单的日期，如5号 |
| **还款日** | ✅ | 1-28 | - | 每月还款截止日，如25号 |
| 信用额度 | ⭕ | 数字 | - | 信用卡总额度，用于计算额度使用率 |
| 最低还款比例 | ⭕ | 百分比 | 10 | 最低还款占账单金额比例 |
| 逾期日利率 | ⭕ | 数字 | 0.05 | 万分之五，超出免息期后的日利率 |
| 免息期（自动计算） | 只读 | 展示 | - | 自动计算最短/最长免息天数 |

> ⚠️ **账单日约束**：还款日必须 ≠ 账单日，且间隔至少 5 天。若用户选择冲突值，前端给出警告提示。

### 免息期计算公式

```
最短免息期（天）= 还款日与账单日同月时（还款日 - 账单日）
最长免息期（天）= 最短免息期 + 约30天

若还款日 < 账单日（还款日在下个月）：
  最短免息期 = (28/30/31 - 账单日) + 还款日
  最长免息期 = 最短免息期 + 约30天

示例：
  账单日5号，还款日25号 → 最短20天，最长50天
  账单日20号，还款日10号 → 最短20天，最长50天
  账单日1号，还款日28号 → 最短27天，最长57天
```

## 三、信用卡账单周期模型

### 周期图解

```
账单日 5号，还款日 25号：

       5月            │          6月               │          7月
  ────────────────────┼───────────────────────────┼────────────────────
                      │                           │
  5/6 ←──────── 消费周期 ──────────→ 6/5          │
       (计入6月5日账单)              出账日        │
                      │                           │
                      6/5 ←── 免息还款期 ──→ 6/25 │
                           20天                   │
                      │                           │
  5/6消费 → 6/25还 → 免息 50天                   │
  6/4消费 → 6/25还 → 免息 21天                   │
                      │                           │
                      6/6 ←──────── 消费周期 ────────→ 7/5
  		       (计入7月5日账单)              出账日
```

### 四个关键时间点

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                        │
│   上期账单日 ──→ 本期账单日 ──→ 本期还款日 ──→ 下期账单日             │
│       │              │              │              │                  │
│       │← 消费周期 →│              │              │                  │
│       │              │←─免息还款期→│              │                  │
│       │              │              │← 宽限期/逾期计息 →│              │
│                                                                        │
│   本期账单日：上期账单日 ~ 本期账单日之间的消费汇总                     │
│   免息还款期：账单日 ~ 还款日之间，在此期间还款免息                     │
│   逾期计息期：还款日后未还部分，按日息万分之五累计                      │
│                                                                        │
└──────────────────────────────────────────────────────────────────────┘
```

### 账单日消费归属规则

| 消费日期 | 归属账单 | 还款日 | 最长免息天数 |
|---------|---------|--------|------------|
| 6月5日（账单日当天） | 计入本期6月5日账单 | 6月25日 | 20天 |
| 6月6日（账单日次日） | 计入下期7月5日账单 | 7月25日 | 50天 |

> 行业惯例：账单日当天的消费通常计入本期或下期取决于银行，本项目统一按"计入本期"处理。

## 四、信用卡还款方式

### 三种还款模式

| 还款模式 | 还款金额 | 利息 | 后续影响 |
|---------|---------|------|---------|
| **全额还款** | = 本期账单总额 | 0（免息） | 下期消费继续享受免息期 |
| **最低还款** | ≥ 账单金额 × 最低还款比例（默认10%） | 未还部分 × 日利率 × 逾期天数 | 剩余金额从消费日起计息，下期消费不再免息 |
| **自定义还款** | 任意金额（≥最低还款额） | 同最低还款逻辑 | 同最低还款 |

### 还款操作页面设计

### 利息计算规则

#### 全额还款 → 利息 = 0

```
无需计算利息，系统自动将还款金额冲抵信用卡余额。
```

#### 最低还款 / 自定义还款 → 按日计息

```
┌─────────────────────────────────────────────────────────────┐
│  逾期利息计算公式（简化为"从还款日次日起计"）：              │
│                                                             │
│  每日利息 = 未还金额 × 日利率                                │
│  累计利息 = 每日利息 × 逾期天数                              │
│                                                             │
│  示例：                                                     │
│    账单 ¥5,000，最低还 ¥500，未还 ¥4,500                     │
│    日利率 0.05%（万分之五）                                  │
│    逾期 10 天后还清                                         │
│    利息 = 4500 × 0.0005 × 10 = ¥22.50                       │
│                                                             │
│  简化说明（记账App采用）：                                  │
│  - 不精确到每笔消费的入账日（计算过于复杂）                  │
│  - 统一从还款日次日起，按未还金额 × 日利率累计               │
│  - 每天自动记账一笔"信用卡利息支出"（或合并为月终汇总）      │
│  - 还清后停止计息                                           │
└─────────────────────────────────────────────────────────────┘
```

#### 全额结清（一次性还清所有欠款）

```
用户可随时选择一次性还清信用卡全部欠款（当前余额的绝对值）。
还清后该信用卡余额变为 0（或正数表示溢缴款）。
此时不产生任何额外利息（之前已累计的除外）。
```

## 五、记账联动规则

### 信用卡消费记账

```
用户消费时选择信用卡账户支付：
  ├── 现金类/银行存款不减少（信用卡支付不立即扣钱）
  ├── 信用卡余额减少（余额变负 = 欠款增加）
  └── 支出分类正常记录
```

### 信用卡还款记账

```
用户还款时：
  ├── 现金类/银行存款减少 = 还款金额
  ├── 信用卡余额增加（负数减小 = 欠款减少）
  │
  ├── 全额还款：
  │   └── 不产生利息支出
  │
  └── 最低/自定义还款：
      ├── 按期出利息计为「信用卡利息支出」
      └── 每天/每月自动生成利息记账记录
```

### 对报表的影响

| 报表类型 | 信用卡消费 | 信用卡还款 | 利息支出 |
|---------|-----------|-----------|---------|
| 支出报表 | ✅ 计入支出分类 | ❌ 不算支出 | ✅ 计入利息 |
| 资产盘点 | 负债增加 | 负债减少 | 净资产减少 |
| 现金流 | ❌ 不扣现金 | ✅ 扣现金 | ✅ 扣现金 |

```typescript
export interface CreditCardFields {

// Account 接口扩展
export interface Account {
  // ... 原有字段

}

// AccountRequest 扩展
export interface AccountRequest {
  // ... 原有字段
```

```
GET /api/accounts/:id/credit-card-bill
```

**响应**：
```json

  "success": true,

    "billingDate": "2026-06-05",

  }
```

```
POST /api/accounts/:id/calc-card-interest
```

**请求体**：
```json
{
  "repaymentAmount": 2000
}
```

```json
{
  "success": true,

    "billAmount": 5000.00,
    "repaymentAmount": 2000.00,
    "unpaidAmount": 3000.00,
    "dailyInterest": 1.50,
```

```
POST /api/accounts/:id/repay-card
```

```json

  "amount": 5000,
```

```
POST /api/accounts/:id/settle-card
```

## 十二、与负债类（贷款）的关键差异对照

| 维度 | 负债类（liability） | 信用卡类（credit_card） |
|------|-------------------|------------------------|
