# 登录页需求文档

## 1. 页面概述

登录页是用户进入「Fire生活家」的第一入口，承载账号认证、新用户引导的核心职责。

## 2. 设计理念

### 2.1 品牌调性
- **Fire生活家** = 温暖 + 财务安全感 + 追求 FIRE 生活方式
- 视觉关键词：温暖、信任、简洁、现代

### 2.2 色彩体系
登录页颜色遵循项目统一主题 Token 体系（详见 `theme-settings/theme.md`）：

| 角色 | CSS Variable | 说明 |
|---|---|---|
| 主色 | `var(--color-primary)` | 按钮、激活态、品牌强调 |
| 主色深 | `var(--color-primary-dark)` | 按钮按下态 |
| 背景色 | `var(--color-bg-page)` | 页面背景 |
| 卡片背景 | `var(--color-bg-card)` | 表单卡片、弹框 |
| 主文字 | `var(--color-text-primary)` | 标题、正文 |
| 次文字 | `var(--color-text-secondary)` | 辅助说明、placeholder |
| 边框/分割线 | `var(--color-border)` | 输入框底线、分隔 |
| 微信绿 | `var(--color-wechat)` | 微信登录按钮（品牌固定） |
| 危险色 | `var(--color-danger)` | 错误提示 |

### 2.3 字体层级
| 层级 | 字号(rpx) | 用途 |
|---|---|---|
| H1 | 44 | 应用名称 |
| H2 | 32 | 按钮文字、Tab 激活 |
| Body | 28 | 输入框、正文 |
| Caption | 24 | 协议文字、辅助链接 |

## 4. 交互规范

### 4.1 Tab 切换
- 两个 Tab：「密码登录」「验证码登录」
- 激活态：文字加粗 + 底部指示条（主色）
- 切换平滑、不闪屏

### 4.2 表单输入
- 手机号：数字键盘，最大 11 位
- 密码：支持显示/隐藏切换
- 验证码：数字键盘，最大 6 位，右侧「获取验证码」按钮，60s 倒计时

### 4.3 按钮状态
- 正常态：主色填充
- 禁用态：灰色（所有必填未完成时）
- 加载态：文字变「登录中...」+ 不可点击

### 4.4 协议弹框
- 点击《用户协议》《隐私政策》弹出底部面板
- 面板高度自适应，最大 70vh，可滚动
- 点击遮罩层或「关闭」按钮关闭

### 4.5 页面进入/退出
- 开屏页 → `reLaunch` → 登录页（清空栈）
- 注册登录成功 → `reLaunch` → 明细首页
- 注册页返回 → `redirectTo` → 登录页

## 5. 功能清单
1. 密码登录（手机号 + 密码）
2. 验证码登录（手机号 + 短信验证码）
3. 微信一键登录（小程序/H5 差异化处理）
4. 用户协议 / 隐私政策查看
5. 跳转注册页
6. 登录成功跳转首页

## 6. 技术要求
- 适配 iPhone SE(375pt) ~ iPhone Pro Max(428pt)
- 使用 rpx 单位，基于 750 设计稿
- 表单校验在客户端完成，减少无效请求
- 密码传输使用 HTTPS + 加盐哈希
